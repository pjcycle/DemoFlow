# DemoFlow SPEC（macOS）

更新时间：2026-05-24  
适用范围：`DemoFlow` 工具集（`录屏 / PiP 摄像 / 屏幕画图 / 视频剪切 / 音频提取（MP3）`）

## 1. 产品定位
`DemoFlow` 是一个 macOS 工具集，包含五大模块：
1. `录屏`
2. `PiP 摄像`
3. `屏幕画图`
4. `视频剪切`
5. `音频提取（MP3）`

本轮目标是：
1. 在不改变录屏与既有导出语义前提下，将 `PiP 摄像` 对齐为“macOS 原生摄像头悬浮子工具”。
2. 将 `录屏` 对齐为“QuickTime 式主屏整屏录制工具”。
3. 新增 `屏幕画图` 独立入口，承载置顶工具条 + 透明画布的桌面标注流程。
4. `视频剪切` 持续保持弹窗式智能裁剪流程。
5. `音频提取（MP3）` 收敛为本地/URL 到 MP3 的可控提取闭环。

## 2. 信息架构（固定）

### 2.1 左侧菜单
左侧菜单固定五项，顺序固定：
1. `录屏`
2. `PiP 摄像`
3. `屏幕画图`
4. `视频剪切`
5. `音频提取（MP3）`

不再提供独立 `麦克风` 菜单。

### 2.2 页面职责
1. `录屏`：主屏整屏录制、录制控制、录制麦克风选择与监听。
2. `PiP 摄像`：悬浮摄像头预览、PiP 专属音视频控制台与 PiP 独立录像质量配置。
3. `屏幕画图`：置顶工具条 + 透明画布 + 桌面标注会话。
4. `视频剪切`：弹窗式智能裁剪入口，优先支持拖拽导入、导入、单视频剪切导出闭环。
5. `音频提取（MP3）`：本地文件/在线 URL 提取 MP3、运行日志、停止、结果定位闭环。

## 3. 控制台布局约束
1. 左侧菜单栏默认启动宽度为 `200 px`。
2. 默认宽度必须在冷启动时强制生效，不受系统分栏历史状态覆盖。
3. 左侧菜单栏允许拖拽调整宽度，最小宽度不低于 `200 px`。

## 4. PiP 摄像模块（本轮强约束）

### 4.1 目标定义
`PiP 摄像` 是工具集中的一个 **macOS 原生摄像头悬浮工具**，用于桌面工作流下提供低延迟、可控、稳定的前置画中画预览体验。

### 4.2 本轮必须提供
1. 悬浮窗始终前置预览（跨 Space / 全屏辅助场景可见）。
2. 视频/音频设备手动选择（支持 iPhone Continuity 场景）。
3. 麦克风预览静音与实时音量电平。
4. 悬浮窗比例切换与自动放缩（`自动 / 16:9 / 4:3`）。
5. PiP 独立录像质量模块与真实生效的码率控制。

### 4.3 本轮边界
1. 允许新增 PiP 页面独立录像入口（开始/停止 + 打开录像文件夹）。
2. 允许新增 `PiP 录像质量` 模块，位置固定在现有预览/录像卡片下方、设备卡片上方。
3. 禁止将 PiP 页面扩展为导出流程工作台。
4. 禁止新增 PiP 页面语音转写入口。

说明：PiP 录像为独立子链路，不改变 `录屏` / `视频剪切` 现有语义。

### 4.4 PiP 录像质量
1. `PiP 录像质量` 模块仅作用于 `PiP` 独立录像输出，不影响主录屏、屏幕画图、视频剪切、音频提取的质量配置、状态通道或持久化键。
2. 视觉样式对齐录屏 `视频画质` 卡片，结构固定为：预设单选列表 + `Custom` 展开区 + `10 分钟预计大小` + 码率提示。
3. 预设固定为：
   - `Small = 2 Mbps`
   - `Balanced = 3 Mbps`
   - `High Quality = 5 Mbps`
   - `Pro Editing = 8 Mbps`
   - `Custom = 默认 3 Mbps`
4. 默认进入页面时选中 `Balanced`。
5. `Custom` 仅允许 `1-30 Mbps`；超出范围立即钳制，并给出可读提示。
6. `< 2 Mbps` 与 `> 8 Mbps` 的码率提示仅在 `Custom` 下显示；预设模式不额外弹警告。
7. PiP 录像状态处于 `preparing / recording / stopping` 时，质量模块整体禁用。
8. `10 分钟预计大小` 固定基于所选视频码率与 `AAC 128 kbps` 音频估算。

### 4.5 PiP 录像写入链路
1. PiP 录像底层固定由 `AVAssetWriter` 直写最终 `mp4`，不再沿用 `.mov` 录制后导出链路。
2. 视频编码固定为 `H.264`，并写入 `AVVideoAverageBitRateKey = selectedMbps * 1_000_000`，确保码率设置真实作用于输出文件。
3. 音频编码固定为 `AAC 128 kbps`。
4. 现有 PiP 录像语义保持不变：
   - 未显示预览时点击开始会先弹出 PiP 再进入录像
   - 收起/关闭 PiP 会自动停止并落盘
   - 标题后缀 ` - 录像中`

## 5. PiP 交互规范

### 5.1 页面定位
1. `PiP 摄像` 页面是工具控制台，不是通用设置页。
2. 主路径必须围绕“弹出 / 收起悬浮预览”组织。
3. 用户心智是“操作一个桌面悬浮工具”，不是“配置一个录屏附属开关”。

### 5.2 默认行为
1. 默认不自动启动摄像头预览。
2. `enableCameraPiP` 默认值为 `false`。
3. `bootstrap()` 仅做设备刷新与状态同步，不自动 `startPreviewIfNeeded()`。

### 5.3 顶部主入口
`PiP 摄像` 页面右上提供主按钮：
1. 未显示预览：`弹出 PiP 摄像`
2. 已显示预览：`收起 PiP 摄像`

点击规则：
1. 点击 `弹出 PiP 摄像`：内部自动启用 PiP，再调用预览弹出动作。
2. 点击 `收起 PiP 摄像`：仅隐藏预览，不反向关闭内部启用状态。
3. 页面不应把内部启用状态作为主操作暴露给用户。

### 5.4 页面组成
1. 顶部操作区：主按钮 + 当前 PiP 状态。
2. 录像质量区：PiP 独立录像预设、`Custom` 码率与预计大小。
3. 设备区：视频设备、PiP 麦克风、刷新与权限恢复。
4. 监听区：静音、音量、实时电平。
5. 窗口区：`自动 / 16:9 / 4:3` 与当前布局摘要。
6. 诊断区：授权、枚举、发现源、应用内诊断。

### 5.5 比例与自动放缩
1. 比例枚举：`PiPAspectRatio.auto / sixteenByNine / fourByThree`。
2. `16:9` 与 `4:3` 切换沿用“保持高度重算宽度”。
3. `auto` 以可见性优先：
   - 以窗口中心为锚点；
   - 始终收敛在屏幕可见区域；
   - 满足最小尺寸与最大宽占比约束；
   - Space/屏幕变化时自动修正。

### 5.6 设备空状态与诊断
1. 无权限、无设备、设备离线必须给出明确状态文案，不允许静默失败。
2. 提供“刷新设备”“请求权限”“打开系统隐私设置”“运行应用内诊断”。
3. 诊断输出统一：`PASS / BLOCKED` 摘要与授权/枚举信息。

## 6. 窗口与设备技术约束（PiP）

### 6.1 悬浮窗前置
1. `NSPanel` 置顶层级：`mainMenu`。
2. `collectionBehavior` 至少包含：
   - `canJoinAllSpaces`
   - `fullScreenAuxiliary`
   - `moveToActiveSpace`
3. Space 切换、App 激活后执行 `orderFrontRegardless` 兜底。

### 6.2 设备选择
1. 视频设备与 PiP 麦克风设备分开选择。
2. 列表标签支持：`Continuity` / `Offline` / `Built-in`。
3. 枚举策略：
   - 优先 `AVCaptureDevice.DiscoverySession`
   - 空结果时回退 `AVCaptureDevice.devices(for:)`
4. 断连回退：设备离线时自动回退可用设备，并提示用户。

### 6.3 监听语义
1. `PiPAudioPreviewConfig.isPreviewMuted` 与 `previewVolume` 仅影响预览监听。
2. 不改变录制导出音轨语义。
3. 实时电平来自 `cameraEngine.previewAudioLevel`。

## 7. 五模块现状语义（保持不变）

### 7.1 录屏模块
1. 仅支持 `macOS 主屏整屏录制`，不再提供区域录制。
2. 点击开始录屏后隐藏主窗口。
3. 录制中显示独立悬浮控制器，主体为相机图标，可拖动。
4. 悬浮控制器标题栏提供停止方块按钮，点击后结束录屏。
5. 录制控制支持主窗口与悬浮控制器共享状态。
6. 录制麦克风可手选并显示监听电平。
7. 正常停止、启动失败、异常停止均自动恢复主窗口。
8. 录制时优先排除 `DemoFlow` 自身应用窗口；若排除失败，需提示录屏控制窗可能进入画面。
9. 录屏页新增独立 `视频质量` 模块，位置固定在 `录制麦克风` 模块上方。
10. 录屏质量预设固定：`Small / Balanced / High Quality / Pro Editing / Custom`。
11. 默认预设固定为 `Balanced`，参数为 `1080p / 30fps / H.264 / 6 Mbps`。
12. 其他预设固定为：
    - `Small`：`720p / 1080p`，`30fps`，`4 Mbps`
    - `High Quality`：`1080p / 2K`，`30fps`，`10 Mbps`
    - `Pro Editing`：`1080p 60fps / 2K 30fps`，`16 Mbps`
13. `Custom` 参数范围固定：
    - 分辨率：`跟随屏幕 / 720p / 1080p / 2K / 4K`
    - 帧率：`24 / 30 / 60 fps`
    - 编码：`H.264 / HEVC`
    - 视频码率：`1 - 80 Mbps`
14. 音频码率本轮不开放给用户配置，固定为 `AAC 128 kbps`。
16. 开始录屏时锁定当前质量配置；录制中修改 UI 不影响当前录制任务。
17. 文件大小估算统一使用：`10分钟大小 ≈ 视频码率 Mbps × 75 + 10MB`。
18. 自定义码率过低、过高时必须给出可读提示；`1080p` 至少建议 `4 Mbps`。
19. `4K` 仅作为高级自定义能力，不作为默认预设；`16 Mbps` 不足以覆盖 `4K` 高质量录制。

### 7.2 屏幕画图模块
1. 独立子工具，不复用 PiP 或录屏悬浮窗。
2. 快捷键驱动：`⌃⌥1~5` 颜色、`⌘⌥1~6` 工具、`⌘⌥C/H/S/D/A` 会话操作。
3. 提供线/箭头/矩形/圆形/错/对六工具与 1~5 固定配色。
4. 支持清空、展示/收起、画布交互开关（穿透）闭环。
5. 监听 Space/App/屏幕拓扑变化并自动恢复前置与回退。
6. 仅写 `drawStatusMessage`，不覆写录屏或 PiP 状态。
7. 录屏时可作为白名单窗口被包含进录制画面（与 PiP 类似）。
8. 清空/收起均支持消隐动画；动画在当前画布窗口内执行，支持随机与固定效果配置。
9. 支持 `⌘Z` 回退上一笔；仅在画图当前活跃时生效，优先让当前焦点窗口/输入控件处理撤销，不注册为全局热键。
10. 已绘制标注支持直接拖动整笔位置；不新增移动工具，不引入控制点缩放或端点编辑。

### 7.4 PiP 热键补充
1. 新增 `⌘⌥P` 用于 PiP 悬浮预览显示/收起切换。
2. PiP 热键与画图热键共享底层注册器，但动作服务解耦。
3. 热键触发后需执行录屏白名单刷新重试，等待窗口 ID 稳定，确保录屏可见 PiP 变化。

### 7.3 视频剪切模块
1. 新增独立菜单入口 `视频剪切`。
2. 默认进入后自动弹出“智能裁剪”弹窗。
3. v1 范围优先：拖拽导入、文件导入、单视频区间剪切导出。
4. 不改变录屏与 PiP 状态机，不覆写 `statusMessage` / `pipStatusMessage`。

### 7.5 音频提取（MP3）模块
1. 作为独立第 5 模块，输入来源固定为本地文件与在线 URL。
2. 输出格式固定为 `mp3`，默认输出根目录为 `~/Movies/DemoFlow/AudioExtract/`。
3. 输出子目录规则固定：`YYYYMMDD_HHMMSS_<source_tag>/`。
4. 支持开始/停止、运行日志、输出目录打开、最新 MP3 定位。
5. 失败提示统一输出“原因 + 下一步命令”，禁止静默失败。
6. 不覆写录屏/PiP/画图状态通道，不改变其他模块状态机。

## 8. 数据模型/接口（当前实现对齐）
1. `SettingsSection`：`recording / pipCamera / screenDrawing / videoCutting / audioExtract / appSettings`。
2. `PiPAspectRatio`：`auto / sixteenByNine / fourByThree`。
3. `PiPLayoutState`：`normalizedRect + aspectRatio`。
4. `PiPAudioPreviewConfig`：`isPreviewMuted + previewVolume`。
5. `PiPRecordingQualityPreset` / `PiPRecordingQualityConfig`：PiP 独立录像质量预设与自定义码率模型。
6. `AppCoordinator`：
   - `statusMessage`（录屏主状态）
   - `pipStatusMessage`（PiP 独立状态）
   - `pipRecordingQualityConfig`（PiP 独立录像质量配置）
7. `RecordingQualityConfig`：承载录屏预设/自定义分辨率、帧率、编码与视频码率。
8. `PiPPreviewRuntime.startRecording(..., qualityConfig:)`：接收 PiP 独立录像质量配置。
9. `RecordingRequest`：承载主屏录制所需的麦克风、摄像头、录屏质量、PiP 窗口与屏幕画图白名单窗口参数。
10. `VideoCuttingViewModel`：承载导入状态、剪切区间、导出状态，不与录屏/PiP 状态串写。
11. `AudioExtractViewModel`：承载来源模式、提取状态、日志与输出定位，不与录屏/PiP/画图状态串写。

## 9. 非目标
1. 本轮不重构录屏核心状态机。
2. 本轮不改变录屏与 PiP 既有业务语义。
3. 本轮不引入 PiP 页面导出流程工作台与转写入口（允许独立 PiP 录像入口）。
4. 视频剪切 v1 不引入多轨、转场、特效合成。
5. 音频提取模块本轮不引入语音转写与批量任务编排。
6. 音频提取模块本轮不支持运行时下载/解压/安装 yt-dlp。

## 16. 音频提取当前执行语义（补充）
1. 输入支持本地文件（`mp4/mov/mkv/webm/mp3`）与在线 URL（B站/YouTube 主支持）。
2. 输出仅为 `mp3`，默认落在 `~/Movies/DemoFlow/AudioExtract/YYYYMMDD_HHMMSS_<source_tag>/`。
3. `<source_tag>` 规则：本地文件取文件名清洗；URL 优先提取 `BVxxxxxxxxxx` 或 `v=xxxxxxxxxxx`，否则清洗 URL 文本；清洗仅保留字母/数字/`-`/`_`，连续 `_` 合并，长度上限 80。
4. 依赖策略：`ffmpeg/ffprobe` 内置优先；`yt-dlp` 仅支持内置 `yt-dlp_macos.bundle`。
5. 运行时安装策略：禁用运行时下载/解压/安装 yt-dlp，禁用 python 脚本模式。
6. 失败提示固定为“原因 + 下一步命令”，并要求日志可追踪。
7. 提取成功前必须通过三重校验：文件存在、体积大于 0、`ffprobe` 时长大于 0。
8. 停止行为必须可中断当前子进程并回收任务状态。
