# DemoFlow AGENTS 约束（macOS）

更新时间：2026-05-26

## 1. 产品边界（固定）
`DemoFlow` 是工具集，固定五模块：
1. `录屏`
2. `PiP摄像`
3. `屏幕画图`
4. `视频剪切`
5. `音频提取（MP3）`

## 2. 本轮核心目标
本轮目标并行：
1. 在不改变录屏语义前提下，`PiP 摄像` 对齐为“macOS 原生摄像头悬浮子工具”。
2. `录屏` 对齐为“QuickTime 式主屏录制工具（默认全屏，支持区域）”。
3. `屏幕画图` 作为独立模块落地“置顶工具条 + 透明画布 + 会话操作”闭环。
4. `视频剪切` 继续承载弹窗式智能裁剪流程。
5. `音频提取（MP3）` 作为第 5 独立模块，收敛到“本地/URL 提取 + 内置依赖优先 + 可读错误闭环”。

## 3. PiP 模块强约束

### 3.1 产品定义
1. `PiP 摄像` 是独立子工具，不是录屏附属面板。
2. 目标是在桌面工作流中提供低延迟、可控、稳定的前置画中画体验。
3. 页面形态应是“控制台”，不是通用设置堆叠页。

### 3.2 允许能力（仅这些）
1. 悬浮窗始终前置预览。
2. 视频/音频设备手动选择（含 iPhone Continuity）。
3. 麦克风预览静音与实时电平。
4. 比例切换与自动放缩。

### 3.3 页面边界（PiP 页面）
1. 允许新增独立 PiP 录像入口（开始/停止 + 打开录像文件夹），且必须与录屏状态机解耦。
2. 允许在设备选择上方新增独立 `PiP 录像质量` 模块，且仅作用于 PiP 独立录像输出。
3. 禁止把 PiP 页面扩展为导出流程工作台。
4. 禁止新增 PiP 页面语音转写入口。

### 3.4 PiP 录像质量规则
1. 模块位置固定为 `预览/录像` 卡片下方、`设备选择` 卡片上方，视觉样式对齐录屏 `视频画质` 卡片。
2. 预设固定为：`Small 2 Mbps / Balanced 3 Mbps / High Quality 5 Mbps / Pro Editing 8 Mbps / Custom 默认 3 Mbps`，默认选中 `Balanced`。
3. `Custom` 仅允许 `1-30 Mbps`；超出范围立即钳制；模块内展示 `10 分钟预计大小`。
4. `< 2 Mbps` 与 `> 8 Mbps` 的码率提示仅在 `Custom` 下显示。
5. PiP 录像状态处于 `preparing / recording / stopping` 时，质量模块整体禁用。
6. 该配置不得复用或覆盖主录屏、屏幕画图、视频剪切、音频提取的质量配置、状态通道或持久化键。
7. PiP 录像底层固定使用 `AVAssetWriter` 直写 `mp4`；视频 `H.264` 按所选 Mbps 写入，音频固定 `AAC 128 kbps`；不得再走 `.mov` 录制后导出链路。

### 3.5 默认行为
1. `enableCameraPiP = false`。
2. 冷启动不自动弹出 PiP 预览。
3. `bootstrap()` 仅刷新设备与同步状态，不自动启动预览。

### 3.6 顶部主按钮规则
1. 未显示预览时仅显示：`弹出 PiP 摄像`。
2. 已显示预览时仅显示：`收起 PiP 摄像`。
3. 点击 `弹出 PiP 摄像`：内部自动启用并执行预览弹出。
4. 点击 `收起 PiP 摄像`：仅隐藏预览，不自动反勾选内部启用状态。
5. 不再把内部启用状态作为用户主操作心智暴露。

### 3.7 比例规则
1. 比例固定：`自动 / 16:9 / 4:3`。
2. `自动`优先保证可见性与可拖拽性。
3. `16:9 / 4:3` 切换保持高度。

### 3.8 PiP 热键
1. 新增全局热键：`⌘⌥P`（显示/收起 PiP 切换）。
2. 触发后需执行录屏窗口白名单刷新重试，确保录屏成片可见 PiP 变化。
3. 热键不可用时允许前台降级，但必须给出可读状态提示。

## 4. 屏幕画图模块约束
1. `屏幕画图` 为解耦独立功能模块，不依附录屏状态机。
2. 工具条与透明画布同生命周期，支持弹出/收起联动。
3. 快捷键驱动能力固定：`⌃⌥1~5` 颜色、`⌘⌥1~6` 工具、`⌘⌥C/H/S/D/A` 会话操作。
4. 工具固定：线、箭头、矩形、圆形、错、对；颜色固定：1红/2黄/3绿/4蓝/5黑。
5. 支持清空、展示/收起、画布交互开关（穿透）会话闭环。
6. 监听 Space/App/屏幕变化，执行前置恢复与屏幕回退提示。
7. 禁止新增录制入口、导出入口。
8. 清空画布与收起画图需走统一消隐动画管线，动画在当前画布窗口内执行。
9. 消隐动画配置固定支持：`随机 / 固定`，固定效果支持 `碎裂掉落、左→右、右→左、上→下、下→上`。
10. 支持 `⌘Z` 回退上一笔；该能力仅在画图当前活跃时生效，优先让当前焦点窗口/输入控件处理撤销，不作为全局热键抢占其它应用的标准撤销。
11. 已绘制标注支持直接拖动整笔位置；不新增第七个移动工具，不引入控制点缩放/端点编辑。

## 5. 状态隔离约束
1. 录屏状态文案仅写 `statusMessage`。
2. PiP 状态文案仅写 `pipStatusMessage`。
3. 屏幕画图状态文案仅写 `drawStatusMessage`。
4. 音频提取状态文案仅写 `AudioExtractViewModel.statusMessage`。
5. PiP/画图/音频提取操作禁止覆写录屏主状态文案。

## 6. 窗口与设备约束
1. PiP 窗口需支持 `canJoinAllSpaces + fullScreenAuxiliary + moveToActiveSpace`。
2. 屏幕画图工具条与画布同样需支持以上跨 Space 行为。
3. Space/App 激活变化后需 `orderFrontRegardless` 兜底。
4. 设备枚举优先 DiscoverySession，空时回退 legacy API。
5. 设备断连不崩溃，需自动回退并给出可读提示。
6. 无权限/无设备禁止静默失败。

## 7. 录屏模块约束
1. 录屏支持 `全屏` 与 `区域` 两种模式，默认模式为 `全屏`。
2. 设置页不再展示 `整屏/区域` 切换按钮；录制入口文案固定为 `弹出录屏控件`。
3. 区域录屏入口固定在悬浮录屏控件中（标注按钮前的虚线框图标）；点击后进入区域框选。
4. 区域框选层支持框内拖动、边缘/角点缩放；远离框体区域保持鼠标穿透，不遮挡录屏控件。
5. 悬浮录屏控件顺序固定为：`时间 / 尺寸 / 区域按钮 / 标注 / 录屏(开始/停止) / 暂停(暂停/继续) / 关闭`。
6. 悬浮控件尺寸显示需实时联动：全屏显示当前屏幕尺寸，区域显示当前框选尺寸；开始录制后锁定当次录制尺寸。
7. 点击开始录屏后立即隐藏主窗口。
8. 录制中通过独立悬浮控制器承载控制操作，不复用 PiP 或画图窗口。
9. 正常停止、启动失败、异常停止都必须自动恢复主窗口。
10. `SCContentFilter` 需优先排除 `DemoFlow` 自身应用窗口；若排除失败，需给出可读提示。
11. 屏幕画图窗口（工具条/画布）可作为白名单窗口加入录屏（类似 PiP 窗口）。
### 7.1 录屏质量 / 码率约束
1. `录屏` 页允许新增独立 `视频质量` 模块，位置固定在当前 `录制麦克风` 模块上方，不改变录屏状态机语义。
2. 录屏质量预设固定：`Small / Balanced / High Quality / Pro Editing / Custom`。
3. 默认选中 `Balanced`，参数固定为：`1080p / 30fps / H.264 / 6 Mbps`。
4. 其他预设参数固定为：
   - `Small`：`720p / 1080p`，`30fps`，`4 Mbps`
   - `High Quality`：`1080p / 2K`，`30fps`，`10 Mbps`
   - `Pro Editing`：`1080p 60fps / 2K 30fps`，`16 Mbps`
5. `Custom` 参数范围固定为：
   - 分辨率：`跟随屏幕 / 720p / 1080p / 2K / 4K`
   - 帧率：`24 / 30 / 60 fps`
   - 编码：`H.264 / HEVC`
   - 视频码率：`1 - 80 Mbps`
6. 音频码率本轮不开放给用户配置，固定为 `AAC 128 kbps`。
7. 开始录屏时必须锁定当前质量配置；录制中修改 UI 不影响当前进行中的成片参数。
8. 文件大小估算规则固定：`10分钟大小 ≈ 视频码率 Mbps × 75 + 10MB`。
9. 自定义码率过低、过高时必须给出可读提示；`1080p` 至少建议 `4 Mbps`。
10. `4K` 仅作为高级自定义能力，不作为默认预设档位；`16 Mbps` 不足以覆盖 `4K` 高质量录制。

## 8. 模块边界约束
1. `录屏` 模块继续承载录制控制与录制麦克风。
2. `PiP 摄像` 继续承载悬浮预览、PiP 音视频控制与 PiP 独立录像质量。
3. `屏幕画图` 承载桌面标注会话，不改变录屏与 PiP 语义。
4. `视频剪切` 模块承载弹窗式快速导入、区间剪切、导出闭环。
5. `音频提取（MP3）` 模块承载本地/URL 到 MP3 的提取闭环，不改变录屏/PiP/画图/视频剪切语义。
6. PiP、画图、音频提取迭代不得改变录屏状态机语义。

## 9. 控制台布局约束
1. 左侧菜单栏默认启动宽度为 `200 px`。
2. 启动时必须强制应用默认宽度，不受历史分栏状态干扰。
3. 左侧菜单栏支持拖拽调整宽度，最小宽度不低于默认值。

## 18. 音频提取（MP3）模块约束
1. `音频提取（MP3）` 为独立第 5 模块，禁止耦合录屏/PiP/画图/视频剪切状态机。
2. 输入来源固定：本地文件（`mp4/mov/mkv/webm/mp3`）与在线 URL（B站/YouTube 主支持，其他 best effort）。
3. 输出格式固定：`mp3`。
4. 默认输出根目录固定：`~/Movies/DemoFlow/AudioExtract/`。
5. 子目录规则固定：`YYYYMMDD_HHMMSS_<source_tag>/`。
6. `<source_tag>` 规则固定：
   - 本地文件：取文件名并清洗；
   - URL：优先提取 `BVxxxxxxxxxx` 或 `v=xxxxxxxxxxx`，否则对 URL 文本做清洗；
   - 清洗规则：仅保留字母/数字/`-`/`_`，其余替换为 `_`，连续 `_` 合并，上限 80。
7. 失败提示统一格式：
   - `原因: ...`
   - `下一步命令: ...`
8. 依赖策略固定：
   - `ffmpeg/ffprobe` 使用发布包内置资源优先；
   - `yt-dlp` 仅支持内置 standalone bundle（`yt-dlp_macos.bundle`）；
   - 禁止运行时下载/解压/安装 yt-dlp；
   - 禁止 `/usr/bin/env python3 yt-dlp` 脚本模式。
9. 结果校验固定三项：文件存在、文件大小大于 0、`ffprobe` 时长大于 0。
10. 禁止静默失败，必须给可读状态与日志。
11. UI 的 `installDependencies` 当前仅保留兼容位，不承诺真实安装行为。
12. 停止能力必须可中断当前子进程并回收任务状态。
